# HTML Semantic Layout

## Overview
Semantic layout uses meaningful tags to describe sections of a page instead of relying only on generic containers.

Common layout elements:

- `<header>`
- `<nav>`
- `<main>`
- `<section>`
- `<article>`
- `<aside>`
- `<footer>`

## Example

```html
<header>
  <h1>Learning Web Basics</h1>
</header>
<main>
  <section>
    <h2>HTML</h2>
    <p>Structure your page clearly.</p>
  </section>
</main>
<footer>
  <p>Copyright 2026</p>
</footer>
```

## Why semantic layout matters
- Improves readability
- Helps accessibility tools
- Makes maintenance easier
- Gives content clearer meaning

## Best practices
- Use semantic tags where they match the content.
- Keep heading structure logical.
- Do not use semantic tags only for styling.
